------------------------------------------------------------------------
MIXED PRECISION (AMP)
------------------------------------------------------------------------
Training the DETR decoder in float32 is ~2x slower than necessary.
torch.amp.autocast + GradScaler gives near-identical accuracy at
half the memory and roughly double the throughput.

  Steps:
  1. Add config flag:  training.amp: true
  2. In detection_train.py train_one_epoch():
       - Wrap the forward + loss in torch.amp.autocast("cuda")
       - Use torch.amp.GradScaler for loss.backward() and
         optimizer.step()
  3. Keep eval in float32 (autocast off) for stable mAP numbers.
  4. GradScaler handles inf/nan from float16 overflow automatically.

  Edge case: focal loss sigmoid can underflow in fp16 on rare
  occasions. If training becomes unstable, wrap just the focal
  loss computation in a float32 context.

------------------------------------------------------------------------
RESUME FROM CHECKPOINT
------------------------------------------------------------------------
If training crashes or is preempted at epoch N, everything restarts
from epoch 1. Saving and loading full training state fixes this.

  What to save (extend the periodic checkpoint):
    {
      "epoch": epoch,
      "head_state_dict": model.head.state_dict(),
      "optimizer_state_dict": optimizer.state_dict(),
      "scheduler_state_dict": scheduler.state_dict(),
      "best_ap": best_ap,
      "config": cfg.to_dict(),    # reproducibility
    }

  What to add to config / CLI:
    training:
      resume: null   # path to checkpoint to resume from

    --resume checkpoints/checkpoint_epoch_30.pth

  In detection_train.py main():
    - If resume path is set, load the checkpoint
    - Restore head weights, optimizer state, scheduler state, best_ap
    - Set start_epoch = checkpoint["epoch"] + 1
    - Continue the training loop from start_epoch
