DATA AUGMENTATION  [DONE]
------------------------------------------------------------------------
Implemented in detection/transforms.py.
Config-driven via augmentation section in default.yaml.
Training datasets get augmentation, eval datasets get plain resize+normalize.

  Available transforms (toggle in YAML):
  - horizontal_flip: true/false   (default: true)
  - color_jitter: true/false      (default: false)
  - multiscale: [list of sizes]   (default: null = disabled)
  - random_crop: true/false       (default: false)
  - crop_min_scale: float         (default: 0.5)

  Still possible to add later:
  - Mosaic / MixUp (aggressive, needs custom collate_fn)

------------------------------------------------------------------------
MIXED PRECISION (AMP)  [DONE]
------------------------------------------------------------------------
Implemented in detection_train.py. Enabled by default (training.amp: true).

  - Forward + loss wrapped in torch.amp.autocast("cuda")
  - GradScaler handles backward, unscale, step, update
  - Grad clipping calls scaler.unscale_() before clip_grad_norm_
  - Eval stays in float32 (no autocast)
  - Disabled automatically on CPU (device != cuda)
  - Toggle: --amp / --no-amp CLI flags, or training.amp in YAML

  If focal loss becomes unstable in fp16, wrap the sigmoid_focal_loss
  call in a torch.float32 autocast context as a targeted fix.

------------------------------------------------------------------------
RESUME FROM CHECKPOINT  [DONE]
------------------------------------------------------------------------
Implemented in detection_train.py + config.py + default.yaml.

  Periodic & best checkpoints now save full training state:
    epoch, head_state_dict, optimizer_state_dict, scheduler_state_dict,
    best_ap, scaler_state_dict (when AMP enabled).

  Usage:
    python detection_train.py --config configs/default.yaml \
        --resume output/detection/checkpoint_epoch_30.pth

    Or in YAML:
      training:
        resume: output/detection/checkpoint_epoch_30.pth

  On resume: head weights, optimizer, scheduler, GradScaler, start_epoch,
  and best_ap are all restored. Training continues from epoch N+1.
